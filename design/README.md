# Design documents

This directory holds feature and subsystem design documents for `dashplayrs`.

Each document describes a proposed change before (or alongside) implementation: motivation,
current gaps, target behaviour, module boundaries, API impact, rollout phases, and test plan.

Documents are intended to be reviewable in isolation. For the overall player architecture,
see [`ARCHITECTURE.md`](../ARCHITECTURE.md). For delivery tracking, see [`ROADMAP.md`](../ROADMAP.md).
